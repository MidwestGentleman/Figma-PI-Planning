# Asset Preparation Guide for PI Planning Templates Plugin

This guide helps you create professional visual assets for your Figma plugin listing.

## Required Assets

### 1. Plugin Icon (128 x 128 pixels)

**Purpose**: The small icon that appears in the plugin browser and user's plugin list.

**Requirements**:
- **Size**: Exactly 128 x 128 pixels
- **Format**: PNG (recommended) or SVG
- **Aspect Ratio**: 1:1 (square)
- **Background**: Transparent or solid color
- **Style**: Simple, recognizable, no text

**Design Tips**:
- Use a simple, bold icon that represents planning or agile methodology
- Consider using:
  - Calendar icon
  - Sprint board icon
  - Planning/roadmap icon
  - Kanban board icon
  - PI/Program Increment icon
- Keep it minimal - details won't be visible at small sizes
- Use high contrast colors
- Test at 64x64 and 32x32 to ensure it's still recognizable

**Color Suggestions**:
- Use your brand colors or Figma's color palette
- Consider: Blue (#18A0FB), Green (#0F9D58), Purple (#7B68EE)
- Ensure good contrast against light and dark backgrounds

**Example Concepts**:
1. Calendar with checkmark
2. Sprint board with cards
3. Roadmap/planning icon
4. PI planning board layout
5. Agile/Scrum icon

### 2. Thumbnail/Cover Image (1920 x 1080 pixels)

**Purpose**: The main image users see when browsing plugins. This is your first impression!

**Requirements**:
- **Size**: Exactly 1920 x 1080 pixels (16:9 aspect ratio)
- **Format**: PNG (for transparency) or JPG
- **Quality**: High resolution, crisp text
- **Content**: Should showcase your plugin in action

**Design Tips**:
- **Show Real Usage**: Use actual screenshots from your plugin
- **Highlight Features**: Include text overlays or annotations pointing to key features
- **Professional Look**: Clean, modern design
- **Readable Text**: If including text, make sure it's large and readable
- **Feature Highlights**: Show 3-5 key features prominently

### 4. Video (Optional)

**Purpose**: Demonstrate the plugin in action.

**Requirements**:
- **Length**: Up to 30 seconds
- **Format**: MP4, MOV, or GIF
- **Size**: 1920 x 1080 pixels (recommended)
- **Content**: Show plugin workflow

**Video Ideas**:
- Quick demo of inserting a template

## Creating Assets in Figma

### Step 1: Create Icon

1. **Create New File** in Figma
2. **Create Frame**: 128 x 128 pixels
3. **Design Icon**:
   - Use simple shapes
   - Keep it minimal
   - Test at smaller sizes
4. **Export**:
   - Select frame
   - Right-click > Export
   - Choose PNG, 1x, 2x, 3x
   - Use 1x (128x128) for plugin icon

### Step 2: Create Thumbnail

1. **Create New File** in Figma
2. **Create Frame**: 1920 x 1080 pixels
3. **Add Screenshot**:
   - Take screenshot of your plugin in action
   - Import into Figma
   - Resize to fit frame
4. **Add Overlays**:
   - Add text for plugin name
   - Add feature highlights
   - Add annotations if needed
5. **Style**:
   - Use consistent fonts
   - Use brand colors
   - Keep it professional
6. **Export**:
   - Select frame
   - Export as PNG or JPG
   - High quality

### Step 3: Create Additional Screenshots

1. **For Each Screenshot**:
   - Create 1920 x 1080 frame
   - Add screenshot or mockup
   - Add annotations if helpful
   - Export as PNG

## Asset Checklist

Before uploading, verify:

- [ ] Icon is exactly 128 x 128 pixels
- [ ] Icon is clear and recognizable at small sizes
- [ ] Thumbnail is exactly 1920 x 1080 pixels
- [ ] Thumbnail shows plugin in action
- [ ] All text is readable
- [ ] All images are high quality
- [ ] Colors are consistent
- [ ] No typos in text overlays
- [ ] Screenshots are up-to-date with current plugin version
- [ ] Playground file (if created) works correctly

## Tools for Asset Creation

### Design Tools
- **Figma**: Best for creating all assets
- **Adobe Photoshop**: For advanced image editing
- **Canva**: Quick templates and designs
- **Sketch**: Alternative design tool

### Screenshot Tools
- **Figma**: Built-in screenshot (Shift + Cmd/Ctrl + 4)
- **macOS**: Screenshot tool (Cmd + Shift + 4)
- **Windows**: Snipping Tool
- **Browser Extensions**: Full page screenshots

### Video Recording
- **Loom**: Easy screen recording
- **OBS Studio**: Free, open-source
- **QuickTime** (macOS): Built-in screen recording
- **Camtasia**: Professional screen recording

## Best Practices

1. **Consistency**: Use consistent colors, fonts, and style across all assets
2. **Quality**: Always use high-resolution images
3. **Simplicity**: Don't overcrowd assets with too much information
4. **Testing**: View assets at different sizes to ensure they look good
5. **Feedback**: Get feedback from others before finalizing
6. **Updates**: Keep assets updated when you update the plugin
